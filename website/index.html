01 / THE PROBLEM

The writer cannot be the checker

Your agent saying
“done” is not evidence.

AI coding agents are excellent generators. They are not independent verifiers of their own work. A green test suite can miss a broken provider schema. A confident reviewer can hallucinate a line that does not exist. A quota outage can look exactly like “zero findings.”

Reviewgate turns completion from a claim into a protocol.

02 / CONTROL FLOW

Inside the agent loop

Four stages.
One hard stop.

01EDIT

Agent writes

Claude Code or Codex edits as usual. Native PostToolUse hooks plus final tree fingerprints capture the change.

02INTERCEPT

Gate captures

When the agent tries to stop, Reviewgate collects the full batch diff.

03CHALLENGE

Panel reviews

Independent models inspect the diff with research context, critic and consensus.

04RESOLVE

Decision required

Fix or reject-with-reason. Blocking findings trigger another review before release.

09 / SECURITY MODEL

Trust boundaries, stated plainly

Strong defaults
start with honesty.

01

Diff sanitisation

Unicode normalisation, injection-marker neutralisation, fenced wrapping, secret redaction and persona reaffirmation.

02

Tamper-evident audit

Every run appends a SHA-256 hash-chained JSONL event that can be verified later.

03

Opt-in denylist sandbox

Seatbelt and bubblewrap restrict writes and mask known secret paths. This is not a host read allowlist.

ALPHA CAVEAT

Sandboxing is opt-in, other non-denied host files may remain readable, and network egress is not isolated. Gemini/agy and OpenCode are agentic CLIs with permission-bypass mode. Prefer your own code and trusted repositories.

11 / FAQ

The short answers

Before you
arm the gate.

What exactly does Reviewgate do?+

It installs native repository hooks into Claude Code, Codex, or both. When files change and the agent tries to stop, Reviewgate sends the collected diff to an independent reviewer panel, aggregates findings and blocks unresolved findings; bounded outage and non-convergence exits are labelled as defer or human escalation, never PASS.

Does Codex require an extra activation step?+

Yes. Installation and activation are separate: init writes .codex/hooks.json, then the user opens Codex /hooks, inspects SessionStart/PostToolUse/Stop and trusts their exact current hash. Codex skips a new or changed hash until it is trusted again.

Can Reviewgate or the coding agent approve Codex hooks for me?+

No. The repository installer and authoring agent are exactly what this checkpoint constrains. Reviewgate can inspect the file and verify its own shims, timeouts and binary, but Codex keeps the user’s hash-trust decision private. Reviewgate never invokes the documented danger bypass.

Why can Doctor show two warnings after a healthy install?+

The Codex warning means hook files are installed but their user-controlled activation cannot be observed by Reviewgate. A separate “reviewer reputation … demoting” warning means local adjudication history placed one reviewer below the configured trust floor; it is calibration, not a hook, login or provider-outage error.

Does it write or commit code?+

No. Reviewgate reports and enforces. The coding agent makes fixes; you still inspect the final diff and commit manually.

Do I need every supported provider?+

No. One working reviewer is enough to start. A heterogeneous panel improves independence and consensus.

Is it really free to run?+

The official Codex, Gemini and Claude CLIs can use your existing subscription quota through OAuth, so there is no separate per-review Reviewgate charge. OpenRouter and some Ollama configurations use API billing.

What happens when a provider times out or hits quota?+

An unsuccessful review is never interpreted as a clean pass. Reviewgate can fail over, track cooldowns, block, or use a bounded and explicitly labelled defer/escalation path.

How does Reviewgate handle false positives?+

It combines deterministic evidence checks, diff scoping, a demote-only critic, reviewer consensus, reputation and a false-positive ledger. A demoted CRITICAL still requires an explicit decision.

Can the agent get stuck forever?+

No. Iterations, cost and repeated signatures are bounded. When Reviewgate cannot converge, it writes ESCALATION.md and hands control to a human instead of looping indefinitely.

What platforms are supported?+

Prebuilt binaries support macOS and Linux on arm64 and x64. Windows users should run it through WSL2.

Is Reviewgate production-ready?+

It is actively dogfooded but still alpha software. Review the security model, start on trusted code, run reviewgate doctor and perform the documented smoke test before relying on it.

Where are findings and decisions stored?+

Under .reviewgate/ as plain Markdown and JSON/JSONL files, including pending.md, decisions, state, audit events and ESCALATION.md.

What is the difference between Lore, Brain, Agent Lessons and the FP Ledger?+

Lore is maintainer-approved project intent injected into reviewers. Brain is curated repository knowledge proposed and validated over time. Agent Lessons teach the coding agent from accepted-and-fixed mistakes. The FP Ledger calibrates reviewers from substantive human rejections. They are separate because each has a different author, consumer and trust boundary.

Can Reviewgate run tests and typechecks before using an LLM?+

Yes. Configure phases.checks with ordered commands. They run fail-fast before the panel; a failure becomes a non-rejectable CRITICAL, while a timeout fails closed.

Does Reviewgate work in Git worktrees?+

Yes, but each worktree is a separate checkout and must be initialized independently. Run reviewgate init inside it. Doctor reports an ungated linked worktree as a failure.

Can I review a plan before implementation?+

Yes. Use reviewgate review-plan with one or more Markdown files for a one-shot adversarial review without entering the code Stop-hook loop.

How do I know a clean pass really happened?+

The gate emits an explicit GATE OPEN or GATE CLOSED completion signal; defer and escalation have their own non-PASS labels. Optional desktop notifications and acknowledgePass make completion visible as well.
